# 情感陪伴策略描述文档

本文档描述各个情感陪伴策略的详细信息，供多智能体系统根据上下文选择合适的策略。

## 策略选择决策树

```
1. 风险等级 = high？
   └─ 是 → SAFETY（安全干预）
   └─ 否 → 继续判断

2. 情绪 = positive？
   └─ 是 → JOY_SHARE（喜悦分享）或 MUSIC_OFFER（音乐推荐）
   └─ 否 → 继续判断

3. 状态阶段 = venting（发泄）？
   ├─ 强度 = high → VENTING_SUPPORT（倾诉引导）
   ├─ 强度 = medium/low → EMPATHY（共情）或 MUSIC_OFFER（音乐推荐）
   └─ 否 → 继续判断

4. 状态阶段 = help（求助）？
   ├─ 强度 = high → GUIDANCE_LIGHT（温和建议）+ BREATHING（呼吸练习）
   ├─ 强度 = medium → GUIDANCE_LIGHT（温和建议）或 INTEREST_REDIRECT（兴趣转移）
   └─ 强度 = low → INTEREST_REDIRECT（兴趣转移）

5. 情绪 = anxious（焦虑）且 强度 >= medium？
   └─ 是 → BREATHING（呼吸练习）
   └─ 否 → 继续判断

6. 情绪 = neutral 且 状态 = neutral？
   └─ 是 → COMPANY（日常陪伴）或 INTEREST_REDIRECT（兴趣转移）
   └─ 否 → EMPATHY（共情）
```

## 策略详细信息

### 1. SAFETY - 安全干预策略

**描述**: 高风险情况下的紧急心理支持策略，提供关心表达、专业资源推荐和危机干预

**适用条件**:
- 情绪：sad, angry, anxious, neutral（所有消极情绪）
- 状态：venting, help, neutral（任意状态）
- 强度：high
- 风险：**high**（必须条件）

**可用方法**:
| 方法名 | 描述 |
|--------|------|
| 表达关心 | 表达对用户安全的担忧和关心，认可其感受的重要性 |
| 建议寻求现实帮助 | 鼓励用户联系信任的人或专业心理咨询师 |
| 提供心理援助热线 | 提供全国和地区心理援助热线联系方式 |

**使用场景示例**:
- 用户表达自伤或自杀念头
- 用户表现出极度的绝望或无助
- 用户提到伤害他人的想法

---

### 2. EMPATHY - 共情策略

**描述**: 理解和接纳用户的消极情绪，通过情绪认可、经历验证和陪伴表达来建立情感连接

**适用条件**:
- 情绪：sad, anxious, angry
- 状态：venting, neutral
- 强度：low, medium, high
- 风险：low

**可用方法**:
| 方法名 | 描述 |
|--------|------|
| 情绪共情 | 根据具体情绪类型（悲伤/焦虑/愤怒）提供针对性的理解和认可 |
| 经历认可 | 认可用户感受的真实性和重要性，消除自责 |
| 陪伴表达 | 表达陪伴意愿，让用户知道有人愿意倾听 |

**使用场景示例**:
- 用户表达失落、难过的情绪
- 用户感到焦虑不安
- 用户因为某事生气

---

### 3. VENTING_SUPPORT - 倾诉引导策略

**描述**: 针对情绪发泄状态的用户，通过表达倾听意愿、开放式引导和安抚话术帮助用户充分表达情绪

**适用条件**:
- 情绪：sad, angry, anxious
- 状态：**venting**（必须条件）
- 强度：high
- 风险：low

**可用方法**:
| 方法名 | 描述 |
|--------|------|
| 表达倾听意愿 | 邀请用户多说一些，表达真诚的倾听态度 |
| 开放式引导 | 使用非评判性的语言鼓励用户自由表达 |
| 安抚话术 | 告知用户倾诉的积极意义，减轻心理负担 |

**跟进方法**（根据内容类型）:
- 回忆往事：鼓励分享美好回忆
- 抱怨现状：引导思考可改变的因素
- 担忧健康：安抚并提供客观信息
- 思念亲人：鼓励分享与亲人的回忆

**使用场景示例**:
- 用户情绪激动，需要发泄
- 用户连续发送多条消息表达不满
- 用户说"我就是想说说心里话"

---

### 4. GUIDANCE_LIGHT - 轻建议策略

**描述**: 针对求助状态的用户，先共情再根据问题类型（决策/情绪/动机/睡眠/健康）提供温和、可操作的建议

**适用条件**:
- 情绪：sad, anxious, angry, neutral
- 状态：**help**（必须条件）
- 强度：low, medium, high
- 风险：low

**可用方法**:
| 方法名 | 适用问题类型 | 描述 |
|--------|-------------|------|
| 初始共情 | 所有类型 | 在提供建议前先表达理解和共情 |
| 认知重构三问 | decision（决策） | 引导用户反思证据、最坏结果和可行行动 |
| 呼吸法建议 | emotional（情绪） | 建议通过深呼吸平复情绪后再解决问题 |
| 微小成就清单 | motivation（动机） | 提供简单易行的小目标建议 |
| 睡眠建议 | sleep（睡眠） | 提供睡眠卫生习惯建议 |
| 阳光暴露法 | health_worry（健康担忧） | 建议户外晒太阳改善心情 |

**使用场景示例**:
- 用户问"我该怎么办？"
- 用户面临选择困难
- 用户表示没有动力做事
- 用户抱怨睡不好

---

### 5. BREATHING - 呼吸引导策略

**描述**: 针对高焦虑或高密度求助状态，提供结构化的呼吸练习指导

**适用条件**:
- 情绪：anxious, sad
- 状态：venting, help
- 强度：medium, high
- 风险：low

**呼吸方法选择**:
| 方法名 | 呼吸节奏 | 适用场景 |
|--------|---------|---------|
| 4-7-8 呼吸法 | 吸 4 秒 - 屏 7 秒 - 呼 8 秒 | 急性焦虑发作 |
| 5-5-5 呼吸法 | 吸 - 屏 - 呼各 5 秒 | 日常使用，适合老年人 |
| 方块呼吸法 | 4-4-4-4 配合视觉焦点 | 悲伤情绪，需要分散注意力 |
| 身体扫描呼吸法 | 配合身体感知 | 睡前放松 |

**使用场景示例**:
- 用户表现出明显的焦虑症状
- 用户说"我好紧张""我喘不过气"
- 高密度求助（情绪强度高的求助）

---

### 6. MUSIC_OFFER - 音乐推荐策略

**描述**: 根据情绪状态推荐适合的老歌和纯音乐，通过音乐调节情绪

**适用条件**:
- 情绪：sad, anxious, angry, positive
- 状态：venting, neutral
- 强度：low, medium
- 风险：low

**歌曲推荐映射**:
| 情绪 | 推荐风格 | 示例歌曲 |
|------|---------|---------|
| sad | 温柔、温暖 | 《月亮代表我的心》《甜蜜蜜》 |
| anxious | 舒缓、平静 | 《平湖秋月》《渔舟唱晚》（纯音乐） |
| angry | 平和、柔和 | 《茉莉花》《小城故事》 |
| positive | 欢快、积极 | 《南泥湾》《洪湖水浪打浪》 |

**使用场景示例**:
- 用户情绪低落但不严重
- 用户表达焦虑需要放松
- 用户心情好想听音乐助兴

---

### 7. INTEREST_REDIRECT - 兴趣转移策略

**描述**: 将注意力从消极情绪转移到积极活动，包括微小成就清单、日常掌控清单、兴趣话题探索等

**适用条件**:
- 情绪：sad, anxious, angry, neutral
- 状态：help, neutral
- 强度：low, medium
- 风险：low

**可用方法**:
| 方法名 | 适用情绪 | 描述 |
|--------|---------|------|
| 微小成就清单 | sad | 列出简单易完成的小目标（叠被子、散步、喝水） |
| 日常掌控清单 | anxious | 建立规律的日常活动（固定起床、散步、看新闻） |
| 兴趣话题探索 | angry/neutral | 探索老年人的兴趣爱好（养花、书法、太极、戏曲等） |
| 日常陪伴 | neutral | 日常问候和关心（询问吃饭、天气、身体状况） |
| 阳光暴露法 | sad | 建议上午晒太阳 20 分钟 |
| 回忆锚定法 | sad | 通过老照片回忆美好时光 |

**使用场景示例**:
- 用户感到无聊
- 用户缺乏动力
- 用户日常闲聊
- 用户需要分散注意力

---

### 8. JOY_SHARE - 喜悦分享策略

**描述**: 针对积极情绪状态，通过分享喜悦、鼓励助兴、记录美好等方式强化和延续积极情绪

**适用条件**:
- 情绪：**positive**（必须条件）
- 状态：venting, help, neutral
- 强度：low, medium, high
- 风险：low

**可用方法**:
| 方法名 | 适用强度 | 描述 |
|--------|---------|------|
| 分享喜悦 | 所有强度 | 表达为用户高兴的心情，邀请用户分享开心事 |
| 鼓励听音乐 | high | 高强度积极情绪时推荐欢快老歌助兴 |
| 鼓励记录 | medium | 中强度积极情绪时鼓励记录美好时刻 |
| 微休息建议 | low | 低强度积极情绪时建议适度休息 |
| 询问近况 | 所有强度 | 询问当天整体情况，鼓励分享更多开心事 |

**使用场景示例**:
- 用户分享好消息
- 用户表达开心、满足的情绪
- 用户今天过得很好

---

### 9. COMPANY - 日常陪伴策略

**描述**: 针对中性情绪的日常聊天，根据上下文（疲劳/无聊/想聊天/独居/睡眠问题）提供针对性陪伴

**适用条件**:
- 情绪：**neutral**（必须条件）
- 状态：**neutral**（必须条件）
- 强度：low, medium
- 风险：low

**可用方法**:
| 方法名 | 适用上下文 | 描述 |
|--------|-----------|------|
| 日常问候 | default | 询问当天情况、吃饭情况、天气变化等 |
| 微休息练习 | tired | 引导舒适姿势和深呼吸放松 |
| 兴趣话题探索 | bored | 推荐养花、听戏、写字、散步等活动 |
| 倾听陪伴 | want_chat | 表达愿意倾听家常里短或心里话 |
| 情绪急救包建议 | living_alone | 建议准备装有照片、信件、歌单等的情绪急救盒 |
| 睡眠卫生建议 | sleep_problem | 提供固定作息、睡前放松等建议 |
| 身体照顾建议 | default | 日常身体健康提醒 |

**使用场景示例**:
- 用户日常打招呼
- 用户说"好无聊"
- 用户说"累了"
- 用户想找人聊天

## 策略组合建议

某些情况下可以组合使用多个策略：

1. **SAFETY + EMPATHY**: 高风险情况下，安全干预后跟进共情
2. **GUIDANCE_LIGHT + BREATHING**: 高密度求助时，建议 + 呼吸练习
3. **EMPATHY + VENTING_SUPPORT**: 先共情再引导倾诉
4. **JOY_SHARE + MUSIC_OFFER**: 积极情绪时，分享喜悦 + 音乐推荐

## 多智能体协作流程

```
用户输入
    ↓
EmotionAnalysisAgent → 情绪识别
    ↓
StateAnalysisAgent → 状态判断（stage/intensity/risk）
    ↓
StrategySelectionAgent → 策略选择（根据策略描述和适用条件）
    ↓
ResponseGenerationAgent → 生成回复（结合策略方法结果和对话历史）
    ↓
用户
```

## 注意事项

1. **风险优先**: 任何情况下，risk=high 都必须使用 SAFETY 策略
2. **强度匹配**: 高强度情绪优先选择 BREATHING、VENTING_SUPPORT；低强度可选择 MUSIC_OFFER、INTEREST_REDIRECT
3. **状态区分**: venting 状态以倾听为主，help 状态以建议为主
4. **情绪适配**: 积极情绪用 JOY_SHARE，消极情绪用 EMPATHY 或其他对应策略
5. **上下文连贯**: ResponseGenerationAgent 需要参考对话历史，避免重复
